# My Cloud

Дипломный проект по профессии «Fullstack-разработчик на Python».

На текущем этапе репозиторий подготовлен как прозрачный каркас под дальнейшую разработку: код, архитектурные заметки, требования и управление задачами разведены по понятным зонам.

## Структура

- `docs/` — база знаний проекта, пригодная для просмотра в Obsidian.
- `docs/netology/` — исходные требования и материалы по заданию.
- `docs/project/` — обзор проекта, roadmap и статус разработки.
- `docs/tasks/` — backlog, текущая работа и выполненные задачи.
- `docs/decisions/` — архитектурные решения в формате ADR-lite.
- `docs/workspace/` — временная рабочая зона для черновиков, заметок и промежуточных файлов.

## Принцип ведения проекта

- В `docs/index.md` хранится точка входа по всему проекту.
- В `docs/project/status.md` фиксируется текущее состояние.
- В `docs/tasks/` задачи разделены на `todo`, `in-progress` и `done`.
- В `docs/decisions/` записываются решения, которые влияют на архитектуру и дальнейшую разработку.
