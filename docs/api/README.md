# API

В этом разделе хранится контракт взаимодействия клиента и сервера.

## Что хранить в этой папке

- список endpoint-ов
- формат запросов и ответов
- DTO и правила валидации
- коды ошибок и их описание
- примеры запросов и ответов
- заметки по версиям API

## Возможные будущие файлы

- `overview.md`
- `auth.md`
- `files.md`
- `errors.md`
- `examples.md`

## Правило для раздела

Если документ нужен для согласования интерфейса между frontend и backend,
его лучше хранить именно здесь.
