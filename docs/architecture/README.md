# Architecture

В этом разделе описываются технические решения проекта. Это уже не требования
курса, а ваш ответ на вопрос, как система будет устроена.

## Что хранить в этой папке

- общую схему компонентов
- описание backend- и frontend-архитектуры
- модель данных и схему БД
- потоки данных между сервисами
- подход к авторизации и правам доступа
- подход к хранению файлов
- архитектурные решения и компромиссы

## Возможные будущие файлы

- `system-overview.md`
- `backend-architecture.md`
- `frontend-architecture.md`
- `database-schema.md`
- `auth-flow.md`
- `file-storage.md`
- `decisions/` для ADR или журналов решений

## Правило для раздела

Если документ объясняет, почему решение принято именно так и как устроена
система, он должен лежать здесь.
